## 《卡片宇宙·产品需求文档（PRD v1.0）》

## 1. 产品概述

“卡片宇宙”（Cardverse）是一个以AI视觉生成、审美策展与版权积累为核心的数字内容平台。

网站旨在通过AI生成工具与人工审美策展机制，为全球用户提供高质量的Quote Card（语录卡片）与城市主题卡片，逐步构建属于创作者的版权资产宇宙。

一句话概括：

**用极致精美的卡片，支撑世界的创作。**

## 2. 功能结构总览

主要模块包括：

首页（瀑布流展示）、分类浏览（侧边栏导航）、搜索与高级搜索、卡片详情页、个人中心、会员体系与付费逻辑、上传与创作系统、后台管理系统（内容审核、用户管理、收益分成）。

## 3. 用户端功能设计

### 3.1 首页（Explore）

整体风格参考 Midjourney 首页（https://www.midjourney.com/explore?tab=video_top)，采用瀑布流布局。

左上角显示网站LOGO与中英文名称,logo在最上面，中文（卡片宇宙）第二行，英文（Cardverse）第三行。

左侧导航栏包含卡片的各种分类按钮（鼠标没有悬停的时候显示白色带灰色描边的圆角矩形按钮，鼠标悬停的时候显示灰色的圆角矩形按钮、可后期编辑），依次为：免费系列、建筑系列、城市系列、诸神系列、国风系列、AI实验室。

左下角为各种功能按钮（与上方按钮样式一致），依次为：中英文切换按钮、价格、个人、帮助、注册、登录（注册和登录按钮要明显较其他按钮为大，登陆后显示为一个按钮，按钮上显示个人账户名称，点击可进入个人页面）。

右侧瀑布流上方为搜索框，搜索框右边为一个高级搜索按钮。

页面主体为卡片缩略展示区域，当鼠标悬停在卡片上，显示略微放大的卡片及显示预览文字，点击后进入卡片详情页。

### 3.2 搜索系统

支持关键词检索（如“国风”、“城市”、“黑色”）。

高级搜索提供多维筛选功能，包括作者或上传者、风格、色系、版权类型等选项。

### 3.3 卡片详情页

每张卡片均拥有独立页面，展示以下功能：

卡片大图显示（带“Cardverse”水印），本系列所有卡片缩略图列表（每一个系列我都会制作类似的10张左右）；文字说明（关键词、作者、卡片介绍、上传时间、下载次数等）；增加文字（重点标注的按钮，点击后可在卡片大图上增加文字，原页面操作，不新弹出页面，点击后在卡片大图右侧增加一列半透明白色按钮，分别为：“字体”“大小”“斜体”“加粗”“半透明”最下方弹出“下载”按钮）；下载按钮（仅下载卡片，不加文字，游客无法下载，高清下载需会员、商业会员或单独购买）；收藏功能；版权说明与授权选项（个人使用或商用授权）；最下方显示推荐系统展示相似风格卡片。

### 3.4 个人中心

个人中心包含：我的头像（图片，支持自动上传头像）、我的账号、我的收藏、我的购买记录、我的会员时长与等级、我的收益（创作者模式启用后可见）、我的上传作品（在作品的右上角有一个单独的“上传按钮”）、下载与授权记录。

### 3.5 上传系统（第四阶段）

上传系统内置于个人中心中，必须登录后才有上传的权限。一次仅支持上传一个系列（最多10张）。

上传时需要：

1.上传图片

2.填写系列名称

3.填写简介

4.选择是否接受商业下载及分润

用户可上传原创或AI生成的卡片作品。系统将自动检测分辨率与风格。

后台审核机制确保上传内容符合“高质量卡片”标准。

创作者可选择是否允许作品被商用、是否参与收益分润。

## 4. 后台管理功能

### 4.1 管理员具备以下权限

用户管理（包括封禁、升级、会员审批）、内容管理（上传、审核、下架、标签）、授权与交易记录查看、版权投诉与违规申诉处理、数据仪表板（包含流量、下载量、收入来源分析等）；后台审核机制采用AI初审与人工复审结合模式，确保视觉一致性与版权合规。

### 4.2 后台角色权限分级

- **管理员（Admin）**：拥有全站管理权限，可审核卡片、调整分类、管理用户、处理举报、发布公告。
- **策展人（Curator）**：负责审核上传内容、编辑卡片文字、推荐优质作品到首页。
- **普通用户（User）**：可浏览、收藏、购买与下载卡片。
- **创作者（Creator）**（二期功能）：可上传作品，查看收益与授权记录。

## 5. 会员体系与授权逻辑

系统包含四种用户等级，分别为游客、会员、商业会员、企业会员和创作者。

1.游客：可以自由浏览网站、截图或收藏预览图，但无下载权限，也无商用权。

2.普通会员：可以下载高清卡片，但仅限个人使用，不具备任何商用授权。

3.商业会员：拥有批量下载和商用授权的权利，可以在购买后用于品牌、出版或设计项目中。

4.企业会员：企业客户可通过后台申请‘企业授权（Enterprise License）’，获得批量下载与API调用权限。

5.创作者：拥有上传与收益分成权限，并可为自己的作品设定是否开放授权。

所有授权均由系统自动生成 PDF 格式的 License 文件，以确保法律有效性。

## 6. 支付与收益系统

### 6.1 网站支付

网站支持多种支付方式，初期使用国内聚合支付（爱发电或Ping++），后期迁移至Stripe/PayPal。

收益结算周期为每月一次。

收益来源主要包括会员订阅、单套卡片授权购买、轻度广告合作以及API调用服务（企业客户）。

### 6.2 支付回调与 License 生成逻辑

1. 用户完成支付后，支付接口（Ping++ / Stripe / 爱发电）返回交易成功回调。
2. 系统验证交易状态并在数据库中生成一条对应的 `Transaction` 记录。
3. 若为成功状态，系统自动生成唯一授权文件（License PDF），包含卡片ID、用户信息、授权编号、时间与签名哈希。
4. License 文件链接写入数据库，并推送到用户个人中心“我的授权”页面。
5. 所有授权操作将触发一次异步日志写入，用于后续防盗追踪与版权核查。

## 7.数据结构草图

### 数据结构草图（Data Model Overview）

核心数据库结构包括以下五个主要数据表：

- **Users**：用户基本信息、会员状态、登录方式、积分记录。
- **Cards**：卡片信息（标题、描述、关键词、生成AI模型、缩略图路径、高清图路径、创建者）。
- **Downloads**：用户下载记录、授权类型、文件ID、时间戳。
- **Transactions**：交易信息（金额、支付方式、状态、回调结果）。
- **Licenses**：授权文件表，记录每张卡片的使用权归属、授权编号、有效期与用户关联。

## 8. 技术架构建议

前端使用 Next.js 和 Tailwind CSS。

后端采用 Supabase 与 Node.js。

数据存储使用 PostgreSQL 和 Cloudflare R2。

图片分发采用 Vercel Edge 或 Cloudflare Images。

登录系统使用 Auth.js，支持多种第三方登录方式。

国际化采用 i18n 实现多语言切换。

## 9. 关键流程

### 9.1 用户路径如下：

新访客：注册→体验→付费→成为会员

会员：访问首页 → 浏览卡片 → 点击详情 → 下载或购买 → 登录 → 支付 → 获得授权文件 → 收藏或分享。

创作者路径为：注册 → 上传卡片 → 系统审核 → 发布 → 用户购买 → 分润结算。

### 9.2 用户旅程图（User Journey）

1. 新访客访问首页，浏览卡片瀑布流。
2. 产生兴趣后，点击单张卡片进入详情页，查看高清预览与文字说明。
3. 选择注册或登录账号（支持邮箱 / 微信 / Google）。
4. 注册后可收藏、创建个人收藏夹。
5. 若想下载高清版本，系统提示开通会员或单次购买。
6. 支付成功后，自动生成授权文件（License PDF），并记录交易。
7. 用户可在“个人中心”查看购买历史、会员时长、收藏与下载记录。
8. 系统定期推荐相似风格或主题的卡片，鼓励二次购买。

## 10. 非功能需求

页面响应时间应小于1秒。

图片加载延迟不超过0.5秒。

网站需兼容主流浏览器与移动端。

支持中英双语切换。

必须具备图像版权追踪与防盗链机制。

为未来接入AI生成模块预留接口。

网站需符合中国及国际数据保护法规，确保用户信息及支付数据加密传输与安全存储。

## 11. 质量标准

所有卡片图片分辨率需不低于1920×1080。

所有卡片应留有部分区域图案简单可在上方写文字。

整体审美风格保持统一，线条清晰，无明显AI伪影或低质痕迹。

每张卡片需附带AI生成说明与原创保证。

前端视觉体验需与 Midjourney 审美保持90%以上一致。

## 12. 关键风险与对策

图片可能被盗用传播，导致版权受损。对策为禁止右键保存、设置防盗链、仅登录后显示高清图。

人工审核成本高、审核难度大。对策为引入AI辅助初审、人工复审机制。

盈利模式可能增长缓慢，存在资金风险。对策为优先建立会员制，再开放授权市场。

用户增长可能停滞。对策为增加社交分享功能、设立每日精选推荐机制。

视觉审美不统一可能削弱品牌识别度。对策为制定并执行统一的视觉风格规范文档（Cardverse Style Guide）。

## 13. 国际化与未来拓展

初期支持中英双语切换，未来计划拓展至日语、韩语、西班牙语和法语等语言。

开放城市卡片与爱卡片API接口，后续将推出‘城市卡片’与‘爱卡片’系列，作为生态子品牌。

支持移动端与第三方合作平台。

未来可与图库网站、AI设计工具实现互通，形成生态闭环。

## 14. 结语

卡片宇宙将AI、美学、版权与收益体系整合为一个轻量却自洽的宇宙系统。
